Sidekick

A self-executing, enclosed namespace providing a main accessor object to the Sidekick.js framework.

Properties

presentation

an object storing key:value pair for references to presentation objects created by createPresentation method. A default 'main' instance is created if no specific naming convention is passed during the createPresentation method. Subpresentations require a unique key to identify them within this property.

Presentation

Presentation object as defined by Sidekick method createPresentation. Returns an accessor object exposing properties and methods allowing control over properties relating to the presentation's structure and slide carousel.

Methods

cycle

parameters
  • direction
  • slideObject
description

cycles the PresentationCarousel in a specific direction. If slideObject argument is passed, will cycle to that slide with the specified direction.

cycleTo

parameters
  • searchphrase
  • callback
description

uses argument searchphrase to find the desired slide within the presentation to cycle to. Automatically cycles to new section if slide is in different section. Primary expected value for searchprase is a slideID as defined within the presentation's JSON data. also supports an index of a slide within the current carousel array, or a reference to a Slide object. Triggers callback if slide cycle results in 'success' if callback is a proper function.

findSlide

parameters

returns
object
description

Searches for a slide either by slideID or by a Slide object reference via argument 'searchprhase', returns a resulting object with specific information about the success of the query, the slide reference, index and section index if successful.

setSection

parameters
  • sectionIndex
  • slideObject
  • fromSwipe
returns
object
description

Sets section to a different section, which changes the PresentationCarousel's slide array. Requires a sectionIndex within the sections array property to switch to. Sections by default are defined in the presentation JSON structure. Optionally can specify a reference to a Slide object within the PresentationSection's slide array to transition to. If a third parameter of 'true' is passed, will transition to the end of the PresentationSection's slide array rather than its start.

PresentationSection

defines a section within the Presentation object. Is automatically created when a section is defined within the presentation structure's JSON data, and is added to the Presentation instance's sections array.

  • slideObject
description

Finds slide object references within the slides array that are adjacent to the slideObject argument. Returns an object with previous/next properties containing references to the adjacent slides.

findSlide

parameters
  • searchstring
description

Finds a reference to a slide as defined by the searchstring argument. searchstring can be either a slideID, an index integer, or a Slide object reference. Returns an object with the found slide's reference and index within the slides array, or a failure report if none is found.

Slide

An object representing an individual slide's properties, access to its HTML wrapper within the carousel wrapper. Many properties are defined by the presentation's JSON data structure, allowing for built-in default properties, user defined properties and custom events for each slide.
